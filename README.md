# VelpTEC Prompt Engineering – Teilprüfung 1

## Overview

This repository contains the practical work completed for Prompt Engineering Teilprüfung 1.

The objective of the exercise was to analyze an existing prompt, identify weaknesses related to efficiency and token consumption, create an optimized version, and compare the generated results.

## Original Prompt

The original prompt requested multiple types of information simultaneously, including:

- Product functions
- Target audience
- Unique characteristics
- Alternatives
- Price information
- Discounts

This resulted in a broad and potentially inefficient prompt that could generate unnecessarily long responses.

---

## Optimized Prompt

The optimized version introduces:

- Maximum response length
- Structured output format
- Reduced ambiguity
- Improved readability
- Lower token consumption

---

## Results

The comparison demonstrated that the optimized prompt:

- Produced shorter responses
- Improved response structure
- Reduced token usage
- Reduced hallucination risk
- Improved clarity and consistency
